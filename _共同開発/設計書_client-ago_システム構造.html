1. 全体アーキテクチャ

【受信フロー】
  公式LINE（ユーザー）
    ↓ Webhook（POST）
  Xserver: system002-od.ordermade-neon.com/line-webhook.php
    ↓ require_once

    ↓ actions に ELVIN_task が含まれる場合

    ↓ VPSからタスク取得→実行
  lineworks_send.py
    ↓ LINE WORKSグループへPlaylwright送信
  ユーザーのLINE WORKS

5. line-webhook.php の処理フロー

1. OPcacheクリア（line-handler.php を強制リロード）
2. api-config.php 読み込み（APIキー・LINEトークン）
3. Webhookボディ受信・署名検証
4. events ループ:
   a. ファイル/画像イベント → save_line_file() でサーバーに保存 → ago_line_msg_cache に格納 → continue
   b. テキスト以外 → continue
   c. テキストイベント:
      i.  「PDFにして」「変換」「ドライブ」等のキーワードを含む場合
          → ago_line_msg_cache から同グループの最新3件のファイルURLを $text に付加
      ii. quotedMessageId がある場合
          → ファイル引用なら save_line_file() してURLを $quoted_text に格納
          → テキスト引用なら $quoted_text にそのまま格納
          → キャッシュ照合フォールバック
      iii.グループメッセージで「ウルバン」「urvan」を含まない → スキップ
      iv. 完了タスク保留通知があれば先にそれを返信して終了
      v.  processLineMessage() に渡す（line-handler.php）
5. 200 OK を返す

6. line-handler.php の処理フロー

processLineMessage($log_entry, $api_key, $line_token):
  1. ユーザー名解決（ago_line_users KVから）
  2. 全業務データ読込（案件・見積・請求・発注・注文・スケジュール）
  3. 会話履歴読込（ago_line_conv_{userId}、最新20件）
  4. 案件サマリー構築（書類URL付き）
  5. システムプロンプト組み立て → Claude Haiku 呼び出し（最大1500トークン）
  6. JSON応答パース（reply + actions）
  7. actionsが空なのに「作業します」系のreplyなら1回リトライ（ELVIN_task強制）
  8. actions を即実行 vs 確認キューに振り分け
  9. 確認キューがあれば菅野社長にLINE PUSH通知
  10. 会話履歴保存（最新10往復）
  11. LINE返信（reply_token で返信）
  12. Web Push通知（管理画面ブラウザへ）
  13. ログ更新

7. AIが認識するアクションパターン（line-handler.php のシステムプロンプト）

パターン指示例AIの動作
パターン1「このPDFをドライブのXXXフォルダに入れて」（ファイル引用あり）ELVIN_task：search_files→create_file(parentId)→notify-completed
パターン2「この画像で見積書作ってドライブに入れてPDFにして送り返して」ELVIN_task：スプレッドシート作成→PDF URL生成→LINEに返信
パターン3「このPDF何が書いてあるか教えて」ELVIN_task：PDFダウンロード→要約→notify-completed
パターン4「この請求書の金額をドライブの集計表に入れて」ELVIN_task：スプレッドシート記入
パターン5「日本ネオンに請求書を送って」ELVIN_task：ago_client_mapで送付先解決→lineworks_send.py→PDF送付
パターン6「ウルバン PDFにして」「さっきのをPDFに」ELVIN_task：直近ファイルをPDF変換してAI事業グループに送付。謝罪・確認返答は絶対禁止
自己処理フェーズ変更・資材ステータス更新・名前登録・スケジュール設定・売上照会ELVIN_taskなし・直接KV更新
ハマりポイント③（パターン6）：ユーザーが引用（リプライ）せずにテキストのみで「PDFにして」と送った場合、AIはファイルを認識できず謝罪返答になっていた。
解決策：line-webhook.php でテキスト処理前に ago_line_msg_cache から同グループの最新3件のファイルURLを $text に付加して渡すようにした（2026-05-28実装）。

8. ELVIN_task の実行フロー

line-handler.php execute_action('ELVIN_task'):
  1. prompt 先頭の「スタッフからの依頼:」を送信者名に強制置換
  2. VPS /api/v1/tasks/recent から直近タスク結果（recent_context）取得
  3. VPS /api/v1/tasks にタスク投入（client_id=ago_001）
     payload: { prompt, requester_id, requester_name, log_id, recent_context }

  ↓ AGO事務所PC の agent_local.py が VPS をポーリング
  ↓ タスク取得 → Claude Code で実行
  ↓ 完了後 notify-completed.php を叩く（結果テキスト送信）

notify-completed.php:
  1. ago_pending_notify_{requester_id} にテキスト保存
  2. 次回ユーザーのLINEメッセージ受信時に先頭で返信（line-webhook.php）
注意：recent_context はELVIN_taskのpayloadに入るが、LINE AIのシステムプロンプトには注入されない。 AIがrecent_contextを見るのはあくまでELVIN_task実行側（agent_local.py）のみ。

10. デバッグ方法

確認したいこと方法
webhookが呼ばれているかXserver ファイルマネージャーで wh_debug.log を確認
AIが何を受け取ったかwh_debug.log の [AI_CALL] 行。text_len でプロンプト長がわかる
AIが何を返したか[AI_RAW] 行（最初の200文字）、[AI_ACTIONS] 行
ELVIN_taskが投入されたか[ELVIN] 行。code=200 なら成功
ファイルキャッシュが注入されたか[CTX_INJECT] 行
デプロイが効いているか[START] webhook called v20260528-XXXX のバージョン文字列で確認
ログの場所：https://system002-od.ordermade-neon.com/ のサーバー上 → Xserverファイルマネージャー → public_html/system002-od/wh_debug.log
